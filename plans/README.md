## Findings considered and rejected

- **PostCSS XSS advisory (SEC-01)**: The vulnerability requires untrusted CSS input. All CSS in this project is developer-authored (Tailwind + shadcn). The fix requires upgrading Next.js, which is a separate migration effort. Not worth doing now — monitor upstream.
- **`dangerouslySetInnerHTML` in chart.tsx (SEC-02)**: Read the full data flow — `ChartConfig` values are developer-authored, no user input reaches this path. Safe as-is. No action needed.
- **Replace framer-motion with CSS transitions**: Bundle savings (~50KB) are real but the effort is M+ and the visual regression risk is MED. The `AnimatePresence` usage for mount/unmount transitions is non-trivial to replicate with CSS. Defer to a future optimization pass.
- **`useMemo` with empty deps in 06-pediatria.tsx**: The underlying data is a module-level constant (never changes). The empty deps array is technically correct for static data. Low confidence it's a real bug. Not worth a standalone fix — addressed incidentally by Plan 007's refactor.
